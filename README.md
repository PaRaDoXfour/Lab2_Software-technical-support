---

## Методологія розробки

### Стратегія гілкування: GitHub Flow
Для управління версіями обрано стратегію **GitHub Flow**, яка базується на використанні головної стабільної гілки (`main`) та короткочасних гілок для кожної нової фічі (`feature/`).

**Процес роботи:**
1. Створення гілки `feature/` від `main`.
2. Регулярні коміти з описом змін.
3. Злиття (Merge) гілки в `main` після завершення етапу.

### Стандарти комітів
Використано систему **Conventional Commits**:
- `feat:` — новий функціонал.
- `style:` — правки стилів та дизайну.
- `docs:` — оновлення документації або контенту.
- `refactor:` — оптимізація коду без зміни логіки.
